# AI 热点自动化 Agent 系统方案

## 一、目标

构建一个低延迟自动化内容系统： 1. 1小时内抓取AI热点 2.
2小时内生成并发布视频 3. 全流程自动化（抓取→分析→生成→发布）

------------------------------------------------------------------------

## 二、总体架构

信息获取 → 分析 → 内容生成 → 视频生成 → 发布 → 数据回流

------------------------------------------------------------------------

## 三、技术选型

### 1. 向量数据库

-   MVP：Chroma（本地部署，低成本）
-   用途：去重 / 聚类
-   数据规模：\< 2000 条，完全够用

------------------------------------------------------------------------

### 2. Agent 框架

选型：LangGraph（轻量使用）

原则： - 只做流程编排（DAG） - 不做自治Agent - 强控制、低延迟

------------------------------------------------------------------------

### 3. 模型选型

-   快速分析：Qwen3.5
-   文案生成：Claude
-   Embedding：轻量模型

------------------------------------------------------------------------

### 4. 基础设施

-   Orchestrator：Go
-   Agent执行：Python
-   队列：RabbitMQ
-   缓存：Redis
-   视频：FFmpeg + TTS

------------------------------------------------------------------------

## 四、Pipeline设计

事件流：

热点抓取 → 去重 → 分析 → 文案 → 视频 → 发布

------------------------------------------------------------------------

## 五、延迟预算

-   抓取：10 min
-   分析：10 s
-   文案：5 s
-   视频：20 min
-   发布：5 min

总计 \< 2小时

------------------------------------------------------------------------

## 六、关键优化

### 1. 并行执行

分析 / 文案 / 素材 同时进行

### 2. 模板化生成

避免自由生成，提高稳定性

### 3. 预热机制

提前准备素材库 / embedding

------------------------------------------------------------------------

## 七、RPA 发布方案

推荐： - Android 模拟器 + Appium（MVP） - 真机农场（长期）

关键点： - 行为模拟（非机械） - 固定设备指纹 - 随机操作节奏

------------------------------------------------------------------------

## 八、风险点

-   平台风控
-   视频生成延迟
-   热点误判

------------------------------------------------------------------------

## 九、MVP实施步骤

1.  抓取Twitter/Reddit，mcp，比如https://6551.io/mcp、https://github.com/GuDaStudio/GrokSearch/tree/grok-with-tavily
2.  Chroma去重
3.  Qwen分析
4.  Claude生成文案
5.  人工发布（初期）

------------------------------------------------------------------------

## 十、核心原则

Agent ≠ 自治系统

而是：

可控流水线 + 状态机 + 低延迟执行
